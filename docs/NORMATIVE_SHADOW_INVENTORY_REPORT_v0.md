# Normative vs Shadow Inventory Report v0

## Purpose

Normative vs Shadow Inventory Report v0 is the machine-generated review report for the PULSE normative vs shadow inventory model.

The report classifies repository workflows, artifacts, tools, docs, reader surfaces, publication surfaces, diagnostic surfaces, binding carriers, and attestation carriers by authority role.

It is a review carrier.

It does not create an independent release-decision engine.

## Builder

The report is generated by:

```text
scripts/build_normative_shadow_inventory_v0.py
```

Example command:

```bash
python scripts/build_normative_shadow_inventory_v0.py \
  --repo-root . \
  --out-json out/normative_shadow_inventory_v0.json \
  --out-md out/normative_shadow_inventory_v0.md
```

## Output artifacts

The builder emits:

```text
normative_shadow_inventory_v0.json
normative_shadow_inventory_v0.md
```

Recommended CI / reviewer paths:

```text
out/normative_shadow_inventory_v0.json
out/normative_shadow_inventory_v0.md
```

## Authority carrier

The PULSEmech authority carrier remains:

```text
status.json
→ declared gate policy
→ workflow-effective materialized required gate set
→ strict fail-closed CI enforcement
```

The inventory report reviews repository surfaces around that path.

## Drift findings

The report may flag drift when:

```text
a workflow is unclassified
a non-authorizing carrier is marked authority-impacting
a reader/publication/diagnostic carrier appears authority-adjacent
a workflow requires explicit carrier-role review
```

Future versions may add deeper drift detection for:

```text
shadow workflow writing release artifacts
publication workflow writing authority artifacts
reader carrier modifying status.json
diagnostic output used as required gate without declaration
mutable action tags in authority / attestation paths
over-broad workflow permissions
```

## Boundary

The inventory report does not change:

```text
PULSEmech decision semantics

required gate wiring
check_gates.py behavior
status schema
CI allow/block behavior
Quality Ledger renderer behavior
```

The report is a review carrier for detecting possible authority drift.
