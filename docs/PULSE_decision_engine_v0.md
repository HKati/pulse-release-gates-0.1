# PULSE Decision Engine v0 — Stability‑Based Release Summary

This document introduces the **PULSE Decision Engine v0**.

The Decision Engine sits **on top of**:

- the deterministic release gates (fail‑closed logic), and
- the derived **Stability Map** artefact (`stability_map.json`, defined in `PULSE_topology_v0.md`).

The goal is **not** to override release decisions, but to:

- read the stability signal from `stability_map.json`,
- derive a small set of **actions** and **risk levels**,
- and produce a structured, human‑readable **decision trace**.

In v0, this file typically contains a single `ReleaseState` under `states[0]`
and an empty `transitions` array. See `PULSE_topology_v0.md` for full schema.

The Decision Engine reads:

- `state.id`
- `state.decision` (`FAIL`, `STAGE-PASS`, `PROD-PASS`, `UNKNOWN`)
- `state.instability.score` (float in `[0, 1]`)
- `state.instability.*` components
- `state.type` (`STABLE`, `METASTABLE`, `UNSTABLE`, `PARADOX`, `COLLAPSE`)
- `state.gate_summary.*`
- optional `state.epf.*` information
