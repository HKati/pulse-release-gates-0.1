# G-shadow pipeline (HPC → overlays → snapshot → governance)

## 1. Inputs: HPC snapshots

- Source: `HPC g_snapshots.jsonl`
- Generated by: [link a releváns HPC folyamatra]
- Granularity: [run / gate / model - amit most használtok]
- Constraints: stable schema, no PII, etc.

## 2. Overlay generation

- Script: `scripts/g_child_field_adapter.py`
- Outputs:
  - `g_field_v0.json`
  - `g_field_stability_v0.json` (diagnostics only)
  - `g_epf_overlay_v0.json` (EPF demo)
  - `gpt_external_detection_v0.json` (GPT usage diagnostics)
- Validation:
  - `schemas/*.schema.json`
  - `scripts/validate_overlays.py` + Overlay schema validation (shadow) workflow

## 3. Snapshot report (shadow)

- Workflow: `G snapshot report (shadow)`
- Inputs: all overlays that exist either in the repo or in the safe pack
- Output artifact:
  - `PULSE_safe_pack_v0/artifacts/g_snapshot_report_v0.md` (zip)
- Sections:
  - Sources (which overlays are present / missing)
  - G-field stats
  - Stability diagnostics
  - EPF panel summary
  - GPT usage summary

## 4. Governance / UX hooks (future work)

- How the snapshot could feed into:
  - release dashboards,
  - gate review processes,
  - audit trails.
- Non‑goals for v0 (explicitly leírni, mit *nem* csinál még).

## 5. Extending the shadow layer

- Adding a new overlay:
  - 1) JSON output → `PULSE_safe_pack_v0/overlays/…`
  - 2) Schema → `schemas/...`
  - 3) Validation hook → `scripts/validate_overlays.py`
  - 4) Snapshot integration → `docs` + report summary
